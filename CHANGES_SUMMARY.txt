================================================================================
                           KEY IMPROVEMENTS
================================================================================

BRIDGE REQUEST HANDLING:
  Before: No timeout, instant failure on bridge down
  After:  8-second timeout, auto-retry with exponential backoff

DUPLICATE DETECTION:
  Before: Database constraint error → 400 Bad Request
  After:  Pre-checked → 409 Conflict (correct HTTP status)

TRANSIENT FAILURES:
  Before: Single error → 500 Server Error
  After:  Auto-retry 2-3 times → success or clear error

ATTENDANCE IDEMPOTENCY:
  Before: Same scan twice → 400 Conflict error
  After:  Same idempotency_key → 200 OK (no error)

ERROR MESSAGES:
  Before: Generic "Server error" with 500 status
  After:  Specific errors with 409/503/400 statuses

================================================================================
                            RETRY STRATEGY
================================================================================

Bridge Client (fingerprint requests):
  - Timeout: 8 seconds
  - Max retries: 2 (total 3 attempts)
  - Backoff: 1s, then 2s
  - Skip retries on 4xx errors

Attendance Marking:
  - Member lookup: 3 retries with 500ms delays
  - Insertion: 2 retries with 500ms delays
  - Idempotent with same key

Member Operations:
  - Lookup: 2 retries with 500ms delays
  - Insert: Pre-check duplicates, then 2 retries
  - Update: 2 retries, validate constraints
  - Delete: 2 retries with 500ms delays

================================================================================
                          HTTP STATUS CODES
================================================================================

200 OK
  - Device check succeeds
  - Attendance marked successfully
  - Bridge offline but gracefully handled (idempotent)
  - Duplicate with same idempotency_key

201 Created
  - Member created successfully
  - Attendance marked for first time

400 Bad Request
  - Missing required fields
  - Invalid action
  - Duplicate attendance (different idempotency_key)

404 Not Found
  - Member not found
  - Device user not found

409 Conflict
  - Phone number already registered
  - Membership number already exists
  - Phone/membership duplicate on update

503 Service Unavailable
  - Bridge offline AND immediate operation (no retry fallback)
  - Bridge timeout after retries

500 Server Error
  - Unexpected database errors
  - Server exception
